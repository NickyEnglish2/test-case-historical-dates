# Тестовое задание: "Ключевые даты"

## Описание

Тестовое задание для демонстрации моих навыков в области Javascript, TypeScript, React. Проект представляет собой интерактивную страницу с историческими датами, реализованную с использованием современных инструментов фронтенд-разработки.

## Технологический стек

### Ядро и сборка

- React 18
- TypeScript
- Webpack 5 (с поддержкой DevServer и HMR)

### Стилизация и анимации

- Styled-components (основная методология стилизации)
- Sass (sass-loader для интеграции стилей)
- GSAP (для реализации сложных анимаций)
- Swiper (для реализации слайдеров)

### Контроль качества кода

- ESLint 9 (Flat Config) — статический анализ кода React и TypeScript
- Stylelint — линтинг стилей внутри styled-components (postcss-styled-syntax)
- Husky — автоматизация Git-хуков
- Commitlint — проверка соответствия сообщений коммитов стандарту Conventional Commits

## Инструкция по развертыванию

### Требования

Для запуска проекта необходимо наличие установленной среды Node.js (рекомендуемая версия LTS) и менеджера пакетов npm.

### Установка зависимостей

Склонируйте репозиторий и выполните команду для установки всех необходимых пакетов:

```bash
npm install
```

### Запуск в режиме разработки

Для запуска локального сервера с поддержкой горячей перезагрузки (Hot Module Replacement):

```bash
npm start
```

После запуска проект будет доступен по адресу: `http://localhost:3000`

### Сборка проекта

Для создания оптимизированной production-сборки в папке `dist`:

```bash
npm run build
```

### Проверка кода (Линтинг)

Для запуска комплексной проверки кода (ESLint и Stylelint):

```bash
npm run lint
```

## Работа с Git

В проекте настроены хуки:

1. **pre-commit**: Автоматически запускает линтеры перед созданием коммита. Если есть ошибки, коммит будет отклонен.
2. **commit-msg**: Проверяет текст сообщения коммита. Сообщение должно соответствовать формату: `type(scope): description` (например, `feat: add historical dates slider`).

## Архитектура

Проект организован по принципу Feature-Based Architecture:

- `src/components` — общие UI-компоненты.
- `src/features` — основные функциональные блоки приложения.
- `src/styles` — глобальные стили и конфигурация темы.
- `src/hooks` — переиспользуемая логика и анимационные хуки GSAP.
